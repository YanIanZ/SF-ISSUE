# Slimefun4 SF1 — SuDo 26.2 + Paper 26.2 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move Slimefun4 onto SuDo `26.2.0` + Paper API `26.2`, migrating every P2a/P2b breaking-change call site so `./gradlew build` is green again.

**Architecture:** SuDo `26.2.0` made its services instance-based behind a `SuDo` context and deleted the static `Scheduler`/`ChatInput`. Slimefun4's `Slimefun` JavaPlugin singleton gains one `SuDo` context (`SuDo.create(this)`) exposed via `scheduler()`/`chat()`/`skins()` accessors; all deleted-static call sites route through `Slimefun.instance().<service>()`. This is a breaking-dependency cutover: bumping the dep breaks all 28 call sites at once, so the migration is atomic — the build is red between the bump and the last migrated site, green only at the end of Task 2.

**Tech Stack:** Java 25, Gradle (Kotlin DSL) + shadow, Paper API 26.2 (compileOnly), SuDo `26.2.0` (implementation, relocated), MockBukkit `mockbukkit-v1.21` 4.41.1, JUnit 5.
